R文件丢失，到处报红，但项目仍可以跑起来

发表于 2017-06-12 | 分类于 Android
| 字数统计 292 | 阅读时长 1

问题

项目功能越堆越多，就会发生一些莫名的奇妙的问题。有问题总是有原因的嘛，所以今天就说一下，R文件丢失，项目竟然能跑起来，而且还到处报红，这对于一个强迫症而言，简直不能忍。
最主要的还是ctrl+鼠标左键不能进行资源文件的跳转，还需要先找布局文件，再copy资源Id去找，不能忍。
效果如下图：

有问题当然是要解决的，接下来是见证奇迹的时刻了

解决

  1. 先找到Android Studio 安卓目录中的idea.properties文件。如图：

  2. 打开这个文件，修改idea.max.intellisense.filesize的值。如图：


    简单的解释一下，这个设定的意思。这个属性对编辑器的文件数量会进行一定的限制，以免编辑器变慢和需要消耗系统更多的内存，你可以删除这个属性或者设置一个你需要的最大值。
  3. 修改后保存文件，重启Android Studio，世界一片美好。如图：
